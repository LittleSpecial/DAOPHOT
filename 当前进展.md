# DAOPHOT 光斑检测算法 - 当前进展与思路

## 项目目标
复眼成像系统的多目标3D定位，核心是准确检测和分离每个子眼区域内的重叠光斑。

## 核心算法流程

```
1. 粗检测: 全图找子眼中心 (min_distance=200)
   ↓
2. 对每个子眼区域:
   - 快速形状检查 (椭圆率>1.3 或 多峰)
   - 若可能重叠 → 高斯拟合分离 (1/2高斯BIC比较)
   - 若单光斑 → 直接质心
   ↓
3. 输出光斑列表 (y, x, amplitude, sigma)
```

## 遇到的问题与尝试

### 问题1: 重叠光斑漏检
**现象**: 双目标图像中很多成对光斑只检测到1个
**原因**: 粗检测的`min_distance=200`会把同一子眼内的2个光斑合并

**尝试的解决方案**:
1. ❌ 在每个子眼内用多高斯拟合 → 太慢 (127秒)
2. ❌ 直接找局部最大值 → 噪声太多 (7000+个)
3. ⏳ 快速预筛查 + 选择性高斯拟合 → **当前测试中**

### 问题2: 误检 (2→3)
**现象**: 有时候把2个光斑识别成3个
**原因**: BIC选择max_n=3时会过拟合

**解决方案**:
- 只比较1高斯和2高斯 (`analyze_overlap_fast`)
- 合并距离<10像素的检测结果
- 过滤振幅<15的弱峰

### 问题3: 速度太慢
**现象**: 对每个子眼都做高斯拟合需要127秒
**目标**: 1-2秒完成

**解决方案**:
- `quick_check_overlap()`: 先用形状分析快速筛查
- 只对可能有重叠的子眼做高斯拟合
- 单光斑直接用质心 (0.1ms vs 900ms)

## 关键函数说明

### `quick_check_overlap(region)`
快速判断区域是否可能有多个光斑:
- 计算二阶矩 → 椭圆率
- 椭圆率 > 1.3 → 可能有重叠
- 或 局部峰值 >= 2 个 → 可能有重叠

### `analyze_overlap_fast(region)`
快速双高斯拟合:
- 只比较1高斯和2高斯 (不尝试3)
- 用BIC选择更优模型
- 限制迭代次数 (max_nfev=100-150)

### `analyze_overlap(region, max_n=3)`
完整版多高斯拟合:
- 尝试1,2,3高斯
- BIC模型选择
- 较慢但更准确
